## 二、稳定性保障措施

### 2.1 通用保障原则

**所有阶段共同遵守**:

| 原则 | 说明 | 执行方式 |
|------|------|---------|
| **向后兼容** | 新功能不影响现有功能 | 新文件新目录，不修改现有文件 |
| **配置开关** | 每个新功能有独立开关 | YAML 配置控制启用/禁用 |
| **灰度测试** | 先测试后上线 | 单只股票测试 → 持仓测试 → 全量 |
| **回滚方案** | 随时可回退到 v1.0 | 备份原文件，一键切换 |
| **日志完整** | 所有操作有日志记录 | 便于问题排查 |

### 2.2 备份策略

**实施前备份**:
```bash
# 1. 代码备份
cp -r ai-invest-system-code ai-invest-system-code.v1.backup.$(date +%Y%m%d)

# 2. 配置备份
cp config/*.yaml config/backup/

# 3. 数据备份
cp -r data/cache data/cache.v1.backup.$(date +%Y%m%d)
cp -r cache cache.v1.backup.$(date +%Y%m%d)
```

**回滚脚本**:
```bash
# 一键回滚到 v1.0
./scripts/rollback-to-v1.sh
```

### 2.3 测试策略

| 测试类型 | 覆盖范围 | 执行时机 |
|---------|---------|---------|
| **单元测试** | 每个新函数 | 开发完成后 |
| **集成测试** | 模块间交互 | 模块完成后 |
| **回归测试** | 现有功能不受影响 | 每次上线前 |
| **灰度测试** | 单只股票 → 持仓 → 全量 | 上线后观察 |

## 三、P0 阶段详细方案

### 3.1 推送优化（仅预警时推送）

**改动范围**:
- 修改：`monitor/holding_monitor.py`
- 新增：`config/push_config.yaml`
- 不改动：其他所有文件

**稳定性保障**:
```yaml
# config/push_config.yaml
# 新增配置，默认关闭新功能
push_optimization:
  enabled: false  # 默认关闭，手动开启
  alert_only_mode: false  # 仅预警推送模式
```

**实施步骤**:
1. 复制原文件为备份：`holding_monitor.py.v1.backup`
2. 新增推送配置项（默认关闭）
3. 修改推送逻辑（条件判断）
4. 单股票测试（华明装备）
5. 持仓测试（2 只股票）
6. 观察 1 天无问题 → 开启全量
7. 保留原文件 7 天，确认无问题后删除

**回滚方案**:
```bash
# 回滚命令
cp monitor/holding_monitor.py.v1.backup monitor/holding_monitor.py
# 重启服务
```

**验收标准**:
- [ ] 预警触发时正常推送
- [ ] 无预警时不推送（日志记录）
- [ ] 现有 Cron 任务不受影响
- [ ] 钉钉机器人配置无需修改

## 六、P3 阶段详细方案

### 6.1 年报深度解读

**改动范围**:
- 新增：`analysis/annual_report_reader.py`
- 新增：`nlp/md&a_analyzer.py`
- 高风险：需要解析 PDF/HTML 年报

**稳定性保障**:
```yaml
# config/annual_report_config.yaml
annual_report_analysis:
  enabled: false  # 默认关闭
  pdf_parser: tesseract  # 或已有 OCR 服务
  analysis_scope: md&a_only  # 仅 MD&A 或全文
  cost_limit_per_report: 5  # 单份报告成本上限
```

**实施步骤**:
1. 新增年报解析模块
2. 测试 PDF 解析准确性
3. 单份年报测试（华明装备 2024 年报）
4. 对比人工阅读结果
5. 验证成本可控
6. 逐步开启

**风险缓解**:
- PDF 解析失败 → 降级为文本摘要
- 成本超限 → 自动暂停
- 分析质量差 → 关闭功能
