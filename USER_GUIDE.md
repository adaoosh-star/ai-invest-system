## 一、系统介绍

### 1.1 系统定位

AI 价值投资系统 v1.0 是基于《投资宪法》（巴菲特/段永平/芒格三位大师的投资哲学）开发的自动化投资分析工具。

**核心定位：**
- ✅ **效率工具** — 帮你做重复的苦力工作
- ❌ **不是决策工具** — 不改变投资逻辑，最终判断权在人工

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **道/术分离** | 《投资宪法》是"道"（不变），AI 系统是"术"（可优化） |
| **AI 是效率工具** | 不预测股价，不牛熊择时，专注企业价值 |
| **回测边界** | 3-5 年长周期验证原则，不是拟合短期收益 |
| **双轨阈值** | 通用底线不变，行业优秀线可调整 |
| **最终判断权在人工** | AI 只提供分析，决策由人工完成 |

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    执行层（Execution）                    │
│  自动化选股 │ 实时监控 │ 预警通知 │ 报告生成              │
├─────────────────────────────────────────────────────────┤
│                    模型层（Model）                        │
│  双轨阈值 │ 事实一致性校验 │ 安全边际动态校准 │ 风险量化  │
├─────────────────────────────────────────────────────────┤
│                    数据层（Data）                         │
│  API 接入 │ 数据清洗 │ 交叉验证 │ 非结构化数据解析        │
└─────────────────────────────────────────────────────────┘
```

### 1.4 技术栈

| 层级 | 技术选型 |
|------|---------|
| 数据源 | Tushare Pro（3100 积分） |
| 编程语言 | Python 3.10 |
| 数据处理 | Pandas + Numpy |

## 三、核心功能

### 3.1 自动化选股（20 项检查清单）

**功能说明：**
批量执行《投资宪法》选股前 20 项检查清单，输出候选股票列表。

**检查项分类：**
| 分类 | 检查项 | 数量 |
|------|--------|------|
| 基本面 | ROE/毛利率/负债率/现金流 | 8 项 |
| 估值 | PE 分位/PB 分位/PEG | 3 项 |
| 行业对比 | 毛利率/ROE vs 行业平均 | 2 项 |
| 现金流质量 | 自由现金流/应收账款/存货 | 3 项 |
| 研发与分红 | 研发费用/分红率 | 2 项 |
| 管理层诚信 | 关联交易/大股东减持/其他应收款 | 3 项 |

**判断标准：**
- ≥15 项✅ → 可买入
- 10-14 项 → ⚠️ 待查
- <10 项 → ❌ 放弃

**使用方法：**
```bash
cd /home/admin/openclaw/workspace/ai-invest-system
python3 selection/checklist_20.py
```

**输出示例：**
```
股票：002270.SZ

详细结果：

  ✅ 毛利率波动：54%±3% (波动 4.0% < 5%)
  ❌ PE 分位：94% (94% > 80%)
  ...

### 3.3 事实一致性校验

3. 对比实际完成数据
4. 计算兑现率
5. 综合评分（0-100 分）

### 3.4 A 股本土化风险量化

**功能说明：**

| 风险 | 判断标准 | 行动 |
|------|---------|------|
| ST/退市 | 是 ST 股或有退市风险 | ❌ 排除 |

| 财务造假 | 审计非标或被立案调查 | ❌ 排除 |
| 流动性 | 日均成交<5000 万 | ⚠️ 关注 |
| 融资盘 | 融资盘>30% | ⚠️ 关注 |
| 北向资金 | 持仓>10% 且 30 天流出>20% | ⚠️ 关注 |

## 六、常见问题

### 6.1 Tushare Token 无效

**问题：** `Token 文件不存在`或`Token 无效`

**解决：**
```bash
# 检查文件是否存在
ls -la ~/.tushare_token

# 检查文件内容
cat ~/.tushare_token

# 重新创建
echo "your_tushare_token" > ~/.tushare_token
chmod 600 ~/.tushare_token
```

### 6.2 数据拉取失败

**问题：** `API 请求失败`或`数据为空`

**解决：**
1. 检查网络连接
2. 检查 Tushare 积分是否足够（≥3100 分）
3. 检查 API 调用频率（Tushare 有限制）

### 6.3 预警通知未推送

**问题：** `钉钉通知未收到`

**解决：**
1. 检查 Cron 任务是否正常运行
2. 检查预警级别（只推送🟠橙色及以上）
3. 检查钉钉推送通道配置

### 6.4 Cron 任务超时

**问题：** `cron: job execution timed out`

**解决：**
1. 检查数据获取是否过慢（交易时间 API 拥堵）
2. 检查是否使用了全市场 API（改用单只股票 API）
3. 增加超时时间或优化数据获取逻辑

### 6.5 输出被截断

**问题：** `钉钉消息显示不完整`

**解决：**
1. 检查脚本是否有过多调试输出（应静默）
2. 确认输出格式为：关键结论 + 完整内容 + 文件路径
3. 大输出应保存到文件，不占用工具通道

### 6.6 选股结果为空

**问题：** `候选股票数量为 0`

**解决：**
1. 检查市场整体估值（可能市场整体偏高）
2. 放宽行业优秀线（通用底线不变）
3. 检查数据是否最新

### 5.2 数据拉取失败

**问题：** `API 请求失败`或`数据为空`

**解决：**
1. 检查网络连接
2. 检查 Tushare 积分是否足够（≥3100 分）
3. 检查 API 调用频率（Tushare 有限制）

### 5.3 预警通知未推送

**问题：** `钉钉通知未收到`

**解决：**
1. 检查 Webhook 地址是否正确
2. 检查预警级别（只推送🟠橙色及以上）
3. 检查钉钉群机器人是否启用

### 6.4 回测结果虚高

**问题：** `回测胜率 100%，感觉不真实`

**解决：**
1. 确认是否包含退市股（改进点 2）
2. 确认回测周期是否≥3 年
3. 确认基准收益是否合理（建议年化 8%）

### 6.5 选股结果为空

**问题：** `候选股票数量为 0`

**解决：**
1. 检查市场整体估值（可能市场整体偏高）
2. 放宽行业优秀线（通用底线不变）
3. 检查数据是否最新
