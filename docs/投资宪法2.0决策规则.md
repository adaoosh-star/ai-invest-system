## 一、决策规则框架

### 1.1 决策类型

| 决策类型 | 触发条件 | 决策结果 |
|---------|---------|---------|
| **买入决策** | 通过 20 项检查 + 双阈值 + 安全边际 | 买入/观望 |
| **卖出决策** | 触发 10 项卖出清单 | 卖出/持有 |
| **补仓决策** | 价格接近补仓位 + 基本面无恶化 | 补仓/持有 |
| **减仓决策** | 价格接近减仓位 + 估值偏高 | 减仓/持有 |
| **持仓决策** | 日常监控 | 持有/预警 |

## 二、买入决策规则（量化）

### 2.1 前置条件（必须全部满足）

```yaml
buy_prerequisites:
  # 1. 20 项检查清单通过率
  checklist_pass_rate_min: 0.75  # ≥75%
  
  # 2. 硬底线检查
  hard_bottom_required: true  # 必须通过
  
  # 3. A 股风险检查
  risk_check_required: true  # 无红色风险
  
  # 4. 流动性要求
  avg_volume_min: 50000000  # 日均成交≥5000 万
```

### 2.2 估值要求（安全边际）

```yaml
buy_valuation:
  # PE 分位要求
  pe_percentile_max: 0.80  # ≤80%
  
  # PB 分位要求
  pb_percentile_max: 0.80  # ≤80%
  
  # PEG 要求
  peg_max: 2.0  # ≤2
  
  # 安全边际折扣
  safety_margin_discount: 0.80  # 8 折买入（合理估值）
```

### 2.3 决策评分

```yaml
buy_scoring:
  # 权重分配
  weights:
    checklist_pass_rate: 0.30  # 30%
    roe_score: 0.25  # 25%
    cash_flow_score: 0.20  # 20%
    valuation_score: 0.15  # 15%
    growth_score: 0.10  # 10%
  
  # 决策阈值
  decision_thresholds:
    strong_buy: 0.85  # ≥85 分：强烈推荐
    buy: 0.70  # ≥70 分：推荐买入
    hold: 0.50  # ≥50 分：观望
    avoid: 0.50  # <50 分：避免
```

## 三、卖出决策规则（量化）

### 3.1 触发条件（满足任意一项即触发）

```yaml
sell_triggers:
  # 1. 基本面恶化
  roe_decline:
    consecutive_years: 2
    decline_threshold: 0.05  # 连续 2 年 ROE 下降>5%
  
  revenue_decline:
    consecutive_years: 2
    decline_threshold: 0.0  # 连续 2 年营收负增长
  
  # 2. 高估
  pe_percentile_sell: 0.95  # PE 分位>95%
  peg_sell: 2.0  # PEG>2
  
  # 3. 风险事件
  st_risk: true  # 被 ST
  pledge_risk: 0.50  # 质押率>50%
  shareholder_reduction: 0.05  # 大股东减持>5%
  
  # 4. 仓位管理
  position_single_max: 0.30  # 单一持仓>30%
  position_industry_max: 0.40  # 单一行业>40%
```

### 3.2 卖出决策评分

```yaml
sell_scoring:
  # 紧急程度
  urgency_levels:
    immediate:  # 立即卖出
      - st_risk = true
      - pledge_risk > 0.50
      - cash_flow_negative_consecutive_2q
    
    soon:  # 尽快卖出
      - pe_percentile > 0.95
      - roe_decline_consecutive_2y
      - revenue_decline_consecutive_2y
    
    consider:  # 考虑卖出
      - position_single > 0.30
      - peg > 2.0
      - gross_margin_decline > 0.05
```

## 四、补仓/减仓决策规则（量化）

### 4.1 补仓决策

```yaml
buy_more:
  # 价格条件
  price_conditions:
    near_buy_position: 0.02  # 现价≤补仓位×1.02
  
  # 基本面条件（必须满足）
  fundamentals_required:
    no_deterioration: true  # 基本面无恶化
    no_red_alert: true  # 无红色预警
  
  # 仓位条件
  position_conditions:
    below_target: true  # 当前仓位<目标仓位
    available_cash: true  # 有可用现金
  
  # 决策规则
  decision:
    if all_met: "补仓"
    if price_only: "准备补仓"
    if fundamentals_bad: "不补仓"
```

### 4.2 减仓决策

```yaml
reduce_position:
  # 价格条件
  price_conditions:
    near_sell_position: 0.02  # 现价≥减仓位×0.98
  
  # 估值条件
  valuation_conditions:
    pe_percentile_high: 0.80  # PE 分位>80%
  
  # 决策规则
  decision:
    if price_and_valuation: "减仓"
    if price_only: "准备减仓"
    if fundamentals_good: "持有"
```

## 五、持仓监控决策规则（量化）

### 5.1 预警级别

```yaml
alert_levels:
  # 红色预警（🔴）- 立即处理
  red:
    - is_st = true
    - pledge_ratio > 0.50
    - pe_percentile > 0.95
    - cash_flow_negative_consecutive_2q
  
  # 橙色预警（🟠）- 密切关注
  orange:
    - shareholder_reduction > 0.05
    - liquidity_risk = true
    - roe_decline > 0.05
    - debt_ratio_rise > 0.20
  
  # 绿色机会（🟢）- 补仓时机
  green:
    - price_near_buy_position
    - pe_percentile < 0.20
```

### 5.2 决策输出

```yaml
decision_output:
  # 推送决策
  push_decision:
    if red_alert: "立即推送"
    if orange_alert: "立即推送"
    if green_opportunity: "立即推送"
    if normal: "不推送（仅日志）"
  
  # 行动建议
  action_recommendation:
    - "立即卖出"
    - "尽快卖出"
    - "考虑卖出"
    - "持有"
    - "准备补仓"
    - "补仓"
    - "准备减仓"
    - "减仓"
```
